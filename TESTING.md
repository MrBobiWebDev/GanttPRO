# 🧪 Тестирование GanttPRO

Этот файл содержит инструкции по тестированию приложения GanttPRO.

## Требования

- Java 17 или выше
- Maven 3.6 или выше

## Автоматические тесты

### Запуск всех тестов

**Windows:**
```bash
mvnw.cmd clean test
```

**Linux/Mac:**
```bash
./mvnw clean test
```

**Или с обычным Maven:**
```bash
mvn clean test
```

### Результат

После выполнения команды вы должны увидеть примерно такой результат:

```

[INFO] BUILD SUCCESS
```

## Ручное тестирование (с браузером)

### 1. Тестирование регистрации

1. Откройте http://localhost:8080/register
2. Заполните форму:
   - Имя: "Test User"
   - Email: "testuser@example.com"
   - Пароль: "test123456"
   - Подтвердить пароль: "test123456"
3. Нажмите "Зарегистрироваться"
4. Вы должны быть перенаправлены на страницу входа с сообщением об успехе

### 2. Тестирование входа

1. На странице http://localhost:8080/login введите:
   - Email: "demo@example.com"
   - Пароль: "demo123"
2. Нажмите "Войти"
3. Вы должны попасть на Dashboard

### 3. Тестирование Dashboard

На странице Dashboard проверьте:
- ✅ Видны статистика (количество проектов и активных задач)
- ✅ Отображаются карточки проектов (должно быть 2 демо-проекта)
- ✅ Кнопка "Создать проект" работает

### 4. Тестирование просмотра проекта

1. Нажмите на один из проектов "Веб-приложение GanttPRO"
2. Проверьте:
   - ✅ Отображается название и описание проекта
   - ✅ Видна диаграмма Ганта слева (задачи)
   - ✅ Видна таблица задач внизу
   - ✅ Разные цвета для разных статусов задач
   - ✅ Прогресс отображается в полосах

### 4.1. Проверка синхронизации диаграммы Ганта

**ВАЖНО**: После исправления бага синхронизации диаграммы Ганта нужно проверить следующее:

#### Горизонтальный скролл (синхронизация дат и задач):

1. На диаграмме Ганта прокрутите содержимое вправо (горизонтальный скролл)
2. Убедитесь, что:
   - ✅ Даты сверху скроллятся вместе с полосками задач (НЕ отдельно)
   - ✅ Полоски задач остаются привязаны к правильным датам
   - ✅ Левая колонка с названиями задач остается неподвижной
   - ✅ При скролле нет рассинхронизации между датами и задачами

3. Прокрутите обратно влево
4. Убедитесь, что диаграмма вернулась в исходное положение без смещений

#### Вертикальный скролл (синхронизация высот строк):

1. На диаграмме Ганта (если много задач), попробуйте скролить вверх/вниз
2. Убедитесь, что:
   - ✅ Названия задач слева и полоски справа скроллятся синхронно
   - ✅ Каждая строка слева соответствует строке справа
   - ✅ Нет пустых промежутков между строками
   - ✅ Высота каждой строки одинакова (60px)

#### Проверка привязки полос к датам:

1. Создайте несколько задач с явно указанными датами:
   - Задача 1: Дата начала: 2026-01-01, Дата окончания: 2026-01-05
   - Задача 2: Дата начала: 2026-01-10, Дата окончания: 2026-01-15
   - Задача 3: Дата начала: 2026-01-03, Дата окончания: 2026-01-08

2. На диаграмме Ганта проверьте, что:
   - ✅ Задача 1 начинается в день 1-го числа
   - ✅ Задача 2 начинается в день 10-го числа (после пробела)
   - ✅ Задача 3 начинается в день 3-го числа
   - ✅ Длина каждой полосы соответствует количеству дней (5, 6 и 6 дней соответственно)

3. Прокрутите диаграмму горизонтально и еще раз убедитесь, что полосы остаются под нужными датами

#### Проверка в режиме Compact (если включен):

1. Если есть режим компактного отображения, включите его
2. Убедитесь, что синхронизация остается корректной в компактном режиме

#### Проверка с фильтрами:

1. Если есть функция фильтрации по статусу, примените фильтр
2. Убедитесь, что:
   - ✅ Скрытые задачи исчезают и слева, и справа
   - ✅ Высоты оставшихся строк совпадают
   - ✅ Диаграмма не оставляет пустые рассинхронизированные промежутки

### 5. Тестирование создания проекта

1. На Dashboard нажмите "+ Создать проект"
2. Заполните форму:
   - Название: "My Test Project"
   - Описание: "This is a test project"
   - Дата начала: выберите дату
   - Дата окончания: выберите дату на 30 дней позже
3. Нажмите "Создать проект"
4. Вы должны вернуться на Dashboard
5. Новый проект должен появиться в списке

### 6. Тестирование создания задачи

1. Откройте только что созданный проект
2. Нажмите "+ Добавить задачу"
3. Заполните форму:
   - Название: "Test Task"
   - Описание: "Test description"
   - Дата начала: текущая дата
   - Дата окончания: дата на 5 дней позже
   - Статус: "В работе"
   - Прогресс: 50
   - Исполнитель: "John Doe"
4. Нажмите "Создать задачу"
5. Вы должны вернуться на страницу проекта
6. Задача должна появиться в таблице и на диаграмме Ганта

### 7. Тестирование редактирования задачи

1. В таблице задач нажмите "Редактировать" для какой-либо задачи
2. Измените статус на "DONE"
3. Нажмите "Сохранить"
4. Прогресс должен автоматически установиться на 100%
5. Цвет полосы на диаграмме Ганта должен измениться

### 8. Тестирование безопасности

1. Откройте браузер с сессией пользователя A
2. На другой вкладке логнитесь как пользователь B (создайте нового пользователя)
3. Попробуйте открыть проект пользователя A по прямой ссылке:
   - URL вроде: http://localhost:8080/projects/1
4. Вы должны получить ошибку доступа или редирект на dashboard

### 9. Тестирование выхода

1. Нажмите кнопку "Выход" в правом верхнем углу
2. Вы должны быть перенаправлены на страницу входа
3. Попробуйте открыть защищенную страницу (например, /dashboard)
4. Вы должны получить редирект на страницу входа

## Решение проблем при тестировании

### Проблема: "Port 8080 is already in use"
**Решение:** Закройте предыдущий процесс или используйте другой порт:
```bash
mvn spring-boot:run -Dspring-boot.run.arguments=--server.port=8081
```

### Проблема: "Tests fail with database connection error"
**Решение:** Убедитесь, что используется H2 база данных (по умолчанию)

### Проблема: "Static files not loading (CSS, JS)"
**Решение:** Перезагрузите браузер (Ctrl+Shift+R или Cmd+Shift+R)

### Проблема: "BCrypt password validation fails"
**Решение:** Убедитесь, что пароли совпадают в форме регистрации

### Как теперь устроен скролл диаграммы:

1. **Горизонтальный скролл:** находится в `.gantt-right-scroll` и скроллит сразу оба элемента (header и rows)
2. **Вертикальный скролл:** синхронизируется между `.gantt-tasks-left` и `.gantt-tasks-rows` через JavaScript
3. **Левая колонка:** остается неподвижной благодаря `flex: 0 0 250px`

### Как рассчитываются left и width:

```javascript
offsetDays = (taskStartDate - ganttTimelineStartDate) / (1000 * 60 * 60 * 24)
left = offsetDays * GANTT_DAY_WIDTH (32px)
width = durationDays * GANTT_DAY_WIDTH (32px)
```

Оба параметра рассчитываются относительно одного `ganttTimelineStartDate`.
