# 🚀 Быстрый старт GanttPRO

## Требования

- **Java 17** или выше
- **Maven 3.6** или выше

## Установка и запуск (Windows/Mac/Linux)

### 1. Откройте терминал в корневой папке проекта

```bash
cd /path/to/GanttPRO
```

### 2. Установите зависимости

```bash
mvn clean install
```

Эта команда скачает все необходимые библиотеки и скомпилирует проект.

### 3. Запустите приложение

```bash
mvn spring-boot:run
```

Приложение запустится на порту **8080**.

### 4. Откройте браузер

```
http://localhost:8080
```

Вы будете перенаправлены на страницу входа.

## Что вы найдете внутри

✅ **Dashboard** - список всех ваших проектов  
✅ **Проекты** - создание и редактирование проектов

## Создание нового проекта

1. Нажмите кнопку **"+ Создать проект"** на Dashboard
2. Заполните форму:
   - Название проекта
   - Описание (опционально)
   - Дата начала и окончания
3. Нажмите **"Создать проект"**

## Добавление задач в проект

1. Откройте проект
2. Нажмите **"+ Добавить задачу"**
3. Заполните форму:
   - Название задачи
   - Описание
   - Дата начала и окончания
   - Статус (Запланирована, В работе, Выполнена, Заблокирована)
   - Прогресс (0-100%)
   - Исполнитель (имя)
4. Нажмите **"Создать задачу"**

## Просмотр диаграммы Ганта

После добавления задач в проект вы сможете видеть диаграмму Ганта прямо на странице проекта:

- **Слева:** список всех задач
- **Справа:** временная шкала с полосами задач
- **Цвета полос:** зависят от статуса задачи
- **Прогресс:** отображается внутри полосы

## Часто задаваемые вопросы

### Q: Приложение не запускается
**A:** Убедитесь, что:
- Установлена Java 17+ (`java -version`)
- Установлен Maven (`mvn -version`)
- Порт 8080 свободен

### Q: Ошибка "command not found"
**A:** Проверьте, что Java и Maven добавлены в PATH:
- Windows: переменная окружения JAVA_HOME
- Linux/Mac: `which java`, `which mvn`

### Q: Потеряны все данные после перезагрузки
**A:** Это нормально, так как используется H2 in-memory база. Для сохранения данных настройте PostgreSQL (см. README.md)

### Q: Как изменить порт?
**A:** Добавьте в `application.properties`:
```properties
server.port=8081
```
